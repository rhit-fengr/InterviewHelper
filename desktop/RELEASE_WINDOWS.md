# Interview AI Hamburger Windows Release Guide

This guide is for producing a Windows `.exe` installer with Electron Builder, signing it, and running final install acceptance checks.

## 1. Prerequisites

- Windows 10/11 build machine
- Node.js 18+ and npm
- Dependencies installed:
  - `cd desktop`
  - `npm install`
- For signed release:
  - Authenticode code-signing certificate (`.pfx`)
  - Certificate password

## 2. Signing Configuration

Electron Builder reads signing credentials from environment variables.

### 2.1 Configure certificate for this shell session

PowerShell:

```powershell
$env:CSC_LINK = "C:\secure\codesign\interview-ai-hamburger-cert.pfx"
$env:CSC_KEY_PASSWORD = "your-pfx-password"
```

Notes:
- `CSC_LINK` can be a local file path or a base64/data URL value.
- Keep cert files and passwords out of git.

### 2.2 Unsigned smoke build (local validation)

```powershell
npm run release:win:unsigned
```

This builds installer artifacts and writes `out/SHA256SUMS.txt` without signature validation.

## 3. Build Commands

### 3.1 Signed release build (recommended for distribution)

```powershell
npm run release:win
```

What it does:
- runs desktop tests
- builds Windows NSIS installer (`x64`)
- generates `out/SHA256SUMS.txt`
- verifies Authenticode signatures for all `.exe` artifacts

### 3.2 Manual build variants

```powershell
npm run build:win
npm run build:win:unsigned
npm run build:win:portable
```

## 4. Expected Artifacts

Output directory: `desktop/out/`

Typical files:
- `Interview-AI-Hamburger-<version>-x64-Setup.exe`
- `Interview-AI-Hamburger-<version>-x64-Setup.exe.blockmap`
- `latest.yml`
- `SHA256SUMS.txt` (generated by release script)

## 5. Signature Verification (manual spot-check)

PowerShell:

```powershell
Get-AuthenticodeSignature .\out\Interview-AI-Hamburger-<version>-x64-Setup.exe | Format-List
```

Optional Microsoft `signtool` check:

```powershell
signtool verify /pa /v .\out\Interview-AI-Hamburger-<version>-x64-Setup.exe
```

Accept only if the signature status is `Valid`.

## 6. Installer Acceptance Checklist

Run these checks on a clean Windows VM or machine:

1. Install flow
- Launch installer, choose install path, complete install.
- Verify desktop/start menu shortcut is created.

2. App startup
- Launch app from shortcut.
- Confirm app opens and reaches Interview Setup screen.

3. Core smoke
- Start server (`server/npm run dev`), then run desktop app.
- Start listening, verify transcript updates.
- Trigger one AI answer using manual input.

4. Settings behavior
- Change font size and verify answer panel reflects it.
- If running packaged Electron app, verify:
  - Always On Top toggle works
  - Window Opacity works
  - Hide App Icon works
  - Hide from Screen Sharing toggle can be switched

5. Undetectable mode smoke
- Enter Undetectable Mode, generate session code.
- Connect mobile client and confirm transcript/answer stream arrives.

6. Upgrade path
- Install a newer version over existing version.
- Confirm app still launches and previous settings remain usable.

7. Uninstall
- Uninstall from Windows Apps/Programs list.
- Confirm install directory and shortcuts are removed.

8. Security checks
- Verify Authenticode signature is valid on final installer.
- Verify `SHA256SUMS.txt` contains hash for each `.exe`.

Release gate:
- No blocker regressions in items 1-8
- Signature valid
- Hash file generated and archived with artifacts
